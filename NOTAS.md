Facturación Electrónica (AFIP) e Impresión de TicketsEstando en Argentina, un sistema de ventas de supermercado no está completo sin la emisión del comprobante fiscal.Backend (Python + AFIP): El backend no solo guarda la venta en PostgreSQL, sino que debe comunicarse con los Web Services de AFIP (WSFEv1).Librería recomendada: Utiliza PyAFIPws en tu entorno de Python. Te abstrae de la complejidad de firmar los XML y manejar los certificados (CRT y KEY).Flujo: Al recibir el POST de la venta desde React, Python arma el comprobante, lo envía a AFIP, espera la respuesta con el CAE (Código de Autorización Electrónico) y el Vencimiento del CAE, y guarda esos dos datos en la tabla ventas antes de hacer el COMMIT final en la base de datos.Frontend (React + Impresora Térmica): Una vez que React recibe el "OK" del backend con el número de ticket y el CAE, hay que imprimirlo. Tienes dos caminos prácticos:Opción WebUSB / Web Serial API: Permite que el navegador (Chrome/Edge) envíe comandos en lenguaje ESC/POS directamente a la impresora térmica conectada por USB. Es la opción más rápida y profesional para puntos de venta.Opción PDF (Fallback): Generar un ticket en formato PDF usando librerías como pdfmake en React y abrir el diálogo de impresión del navegador. Es más fácil de implementar, pero requiere un clic extra del cajero.Modelo Matemático para Alertas de StockPara automatizar la gestión y no depender de la revisión visual, el sistema debe calcular cuándo pedir más mercadería. No basta con un número fijo; depende del flujo de ventas.
Puedes implementar un script en Python (que corra como un proceso cron o background task) que evalúe periódicamente esta fórmula para cada producto:$Stock_{minimo} = (Venta_{diaria\_promedio} \times Tiempo_{reposicion}) + Stock_{seguridad}$ Venta diaria promedio: Promedio de unidades vendidas en los últimos 30 días (sale de una query SQL a la tabla ventas_detalle).Tiempo de reposición: Cuántos días tarda el proveedor en entregar desde que se le hace el pedido.Stock de seguridad: Un colchón extra por si hay demoras o un pico de ventas inesperado.Cuando el stock_actual en PostgreSQL caiga por
